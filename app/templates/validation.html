Backtest / validation

Replay now tests range-cycling behaviour rather than late-session directional continuation. A hit only counts after a post-scan lower-band entry touch and before the two-hours-before-close cutoff.

Three-month historical download, validation, and calibration

This workflow now supports an offset ladder sweep so the app can compare multiple checkpoints, recommend a stronger multi-step live schedule, and only then consider calibration on the best primary checkpoint.

Historical counterfactual gate audit

This audit reruns the same historical period under baseline and relaxed gate scenarios so the app can test whether lower liquidity or price thresholds admit useful extra stage-2 names or just admit junk.

Offline goal-seek optimizer

This walk-forward optimizer replays the scanner as if it were live, searches a bounded set of gate challengers, calibrates weights only on the train windows, and promotes only challengers that improve the out-of-sample goal score across time slices. The focused liquidity scope is the faster default because recent live confirmation packs showed starvation was dominated by liquidity and price gates rather than by scoring weights.
